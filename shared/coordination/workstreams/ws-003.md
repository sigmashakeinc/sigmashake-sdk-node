# Logged-In UI Redesign — Professional Dashboard Experience

## Goal

Fix the broken Security page (`GET /account/security` returns "Error fetch failed") and redesign all logged-in pages (Dashboard, Security, Governance, Logs) with a polished, professional dark-theme UI. Inspired by best-in-class SaaS admin consoles: clean data tables, consistent navigation, status indicators, search/filter patterns, and clear information hierarchy.

## Problem Statement

1. **Security page broken:** `GET https://api.sigmashake.com/account/security` fails with "Error fetch failed" — page shows error state instead of security settings
2. **Logged-in UI lacks polish:** Current pages are functional but need professional-grade UX — consistent layout shell, improved tables, better empty/loading/error states, cohesive navigation
3. **No unified layout shell:** Each logged-in page independently renders its own structure rather than sharing a common authenticated layout

## Design Principles

- Dark theme, high contrast, clean typography
- Data tables with sortable columns, status dots/badges, and row actions
- Consistent top navigation bar with icon+label tabs across all logged-in pages
- Search bars and filter controls where data is listed
- Card-based sections for grouped settings/actions
- Professional loading skeletons, empty states, and error states
- User identity display (email, plan/tier badge) in navigation
- Responsive: works on desktop and tablet

## Reference UI Patterns (from SaaS admin consoles)

- **Top nav bar:** Horizontal tabs with icons — e.g., Dashboard, Agents, Governance, Security, Logs, Settings
- **Data tables:** Column headers (uppercase, small), status indicators (colored dots), action menus (three-dot)
- **Page headers:** Title + subtitle description + primary action button (top-right)
- **Settings pages:** Card sections with clear labels, toggle switches, action buttons
- **Search/filter bar:** Search input + filter dropdown + optional export/download button
